# /log — 作業ログに追記

「logに入れて」「ログに残して」等の指示で呼ぶ。直近の重要タスクの要約を `_worklog/` に残す。

## 動作
1. 対象タスクを特定（`$ARGUMENTS` 指定があればそれ。無ければ直近に実行した重要タスク）。
2. `_worklog/YYYY-MM-DD_<タスク名>.md` に追記（同タスクのファイルが既にあれば追記、無ければ新規）。
3. 中身は人が読める**要約のみ**: 目的 / 主要な意思決定と理由 / ハマり所 / 残タスク。逐語コピーはしない。
4. このログは**起動時に自動ロードしない**。後から見返したいときに該当ファイルだけ読む。

詳細な方針は `_worklog/README.md`。
