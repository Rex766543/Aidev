# /cat — ナレッジをカテゴリ分けして格納

ナレッジファイル（`/ko` 直後のもの、または既存ファイル）を内容・タグから最適なカテゴリへ
振り分け、`knowledge-base/indexes/` を更新するコマンド。`/ko` から呼ばれるほか、単体でも使える。

## 実行手順

`.claude/skills/cat/SKILL.md` のスキルに従って処理する:

1. 対象ファイルを特定（`$ARGUMENTS` にパスがあればそれ。無ければ `knowledge-base/sources/_inbox/`
   の中身を候補提示）。
2. **`knowledge-base/sources/` 直下の実フォルダ一覧を読み取り**、それを現在のカテゴリ集合とする
   （カテゴリ名はハードコードしない。増減・改名に追従するため）。
3. ファイルの `topic` / `tags` / 要約から最適カテゴリを判定。
   - 合致なし → `mix/`。
   - 既存と本質的に違う新領域 → 新カテゴリ作成を AskUserQuestion で提案（承認時のみ作成）。
   - 判定が割れる → 候補（推奨先頭・`mix` 含む）を提示して選ばせる。
4. frontmatter の `topic` を決定カテゴリに更新し、ファイルを `sources/<category>/` へ移動。
5. `indexes/<category>.md` に1行追記（移動元の旧インデックス行があれば除去）。
6. 配置先・更新インデックス・新規カテゴリ有無を報告。
