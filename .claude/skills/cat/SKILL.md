# カテゴリ分けスキル (cat)

## 概要
ナレッジファイル（`/ko` 直後のもの、または既存ファイル）の内容・`topic`・`tags` をもとに、
最適なカテゴリフォルダへ振り分け（移動 or 作成）し、対応する `indexes/<category>.md` を更新する。

## トリガー
- `/ko` 完了後の確認で「はい」を選んだとき
- `/cat` コマンドから単体で呼び出されたとき（既存ファイルの再分類・未分類 `_inbox/` の整理など）

## 入力
- 対象ファイルのパス。指定が無い場合は `knowledge-base/sources/_inbox/` 内のファイルを対象候補として提示する。

## 重要原則：カテゴリは固定しない
カテゴリは将来 増減・改名され得る。**カテゴリ名をハードコードしないこと。**

1. 実行時に必ず `knowledge-base/sources/` 直下の**実フォルダ一覧を読み取り**、それを現在のカテゴリ集合とする
   （`_inbox` は対象外）。
2. 既存カテゴリ以外への対応もこの手順で自然に行える（フォルダが増えていれば自動で候補に入る）。

## 手順
1. **現在のカテゴリを取得** — `knowledge-base/sources/` 直下のフォルダ一覧を `ls` 等で取得（`_inbox` 除く）。
2. **対象ファイルを読む** — frontmatter の `topic` / `tags` と本文の要約・学びを確認。
3. **最適カテゴリを判定**
   - `topic` と `tags` を既存カテゴリ名・各カテゴリの傾向に照らして最も合うものを選ぶ。
   - 明確に合致するものが無ければ `mix/` を既定にする（「迷ったら mix、検索はタグで補う」）。
   - 既存のどれとも本質的に違い、かつ今後も増えそうな新領域だと判断したら、**新カテゴリ作成を提案**する
     （AskUserQuestion）。承認時のみ `sources/<new>/` と `indexes/<new>.md` を新規作成。
   - 判定が割れる場合は AskUserQuestion で候補（推奨を先頭、`mix` を含む）を提示してユーザーに選ばせる。
4. **frontmatter 整合** — 決定カテゴリで `topic` を更新（ファイル名はそのまま）。
5. **ファイル配置** — 対象ファイルを `knowledge-base/sources/<category>/` へ移動。
   - `_inbox/` 起点なら移動。既に別カテゴリにある既存ファイルの再分類なら移動。
6. **インデックス更新** — `knowledge-base/indexes/<category>.md` の `## sources` 以下に1行追記:
   ```
   - YYYY-MM-DD [タイトル](../sources/<category>/ファイル名.md) — タグ: tag1, tag2
   ```
   - 旧カテゴリから移動した場合、旧 `indexes/` の該当行があれば除去する。
7. **報告** — 配置先・更新したインデックス・（あれば）新規作成カテゴリを簡潔に伝える。

## 注意
- このスキルは `sources/` のファイルを対象とする。`syntheses/` の整理に使う場合も同じ手順
  （`syntheses/<category>/` を対象に読み替える）。
- ファイル内容（要約・学び）は書き換えない。frontmatter の `topic` と配置・インデックスのみ扱う。
