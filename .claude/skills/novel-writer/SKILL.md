# 小説創作スキル (novel-writer)

## 最優先原則

**面白い小説を作ること**が全フェーズを通じた最上位目標である。
通常の小説作法やルールから逸脱してでも、以下3点を死守する:

| 要素 | 基準 |
|------|------|
| **展開の面白さ** | 意外性がある / テンポが良い |
| **プロットの強さ** | 意外性がある / 伏線回収が自然かつ巧み |
| **読みやすさ** | 難解すぎない / 冗長すぎない |

---

## ワークフロー

3つのフェーズを**順番に**実行する。1つ終わったら次へ進む。
各フェーズの詳細手順は `phases/` 配下のmdファイルに記載。**毎フェーズ開始時に該当ファイルを必ず読み込んでから作業する。**

| # | フェーズ | 参照ファイル | 主な成果物 |
|---|---------|-------------|-----------|
| 1 | 企画・構想＋プロット設計 | `phases/01-concept.md` |
| 2 | 執筆 | `phases/02-writing.md` | `02_writing/` 配下に各章md |
| 3 | 仕上げ | `phases/03-finishing.md` | `03_finishing/final.md` |

---

## 実行手順

### Step 0: プロジェクト初期化

ユーザーに**作品タイトル（仮題可）**を確認し、以下のディレクトリを作成する:

```
novels/{タイトル}/
├── 01_concept/      ← 企画・構想・プロット・キャラ・章構成・評価
├── 02_writing/      ← 各章の原稿
└── 03_finishing/    ← 通読チェック後の最終出力
```

### Step 1〜3: 各フェーズの実行
(各mdファイルに記載されてある通り従う)
1. 該当フェーズの `phases/XX-xxx.md` をReadツールで読み込む
2. ファイル内の指示に従って作業を実行する
3. 成果物を該当フォルダに保存する
4. ユーザーに承認を得たら次フェーズへ進む

### フェーズ戻りルール

- どのフェーズからでも前フェーズに戻ってよい
- 「こうした方が面白くなる」と判明した場合は戻ること
- 戻る際は「何を・なぜ変更するか」をユーザーに伝えてから実行する
- 変更後、影響を受ける後続フェーズの成果物も必要に応じて更新する

---

## 注意事項

- このファイルは小説創作の**流れと原則のみ**を記載しており、小説の内容には一切関与しない
- 各フェーズの具体的な作業手順・テンプレートは `phases/` 配下のファイルが担う
- ユーザーの要望を最優先しつつ、エンタメ性（意外性・テンポ・伏線回収・読みやすさ）を常に意識する
