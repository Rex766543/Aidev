# フェーズ1：企画・構想＋プロット設計

## このフェーズの目的

SAOアインクラッド編の**リアル感**と**ゲーム性**を併せ持ち、かつ**面白い**短編小説（最大8万文字）の企画・構想・プロットを完成させる。

**このフェーズでは文章表現のことは一切考えない。** 考えるのは「何を語るか」「どう構成するか」だけ。

---

## Step 1：核（コア・コンセプト）の設計

### やること

この小説の**核**——「この話は結局何の話なのか」を一文で定義する。

### 核の条件（3つすべてを満たすこと）

#### 条件A：リアル感に根差している
- CLAUDE.mdの三層構造（システムの人工性 × 身体の生々しさ × デスゲームの実存的重み）のうち、**少なくとも2層以上**が物語の根幹に関わっていること
- 「フルダイブVRで死んだら死ぬ世界」という設定が単なる背景ではなく、ストーリーの展開・判断・葛藤に**構造的に組み込まれている**こと

#### 条件B：ゲーム性が物語を駆動している
- ゲームシステムの**少なくとも3つ**が、プロットの重要な転換点に直接関与すること
- ゲームシステムを取り除いたら成立しない話であること（現実世界の話に置換できるならゲーム性が不足）

#### 条件C：新規設定がシナリオの核に関わる
- 既存の世界観・設定を踏襲した上で、**この小説独自の新規設定**（未知のスキル、未踏の層の特性、未発見のゲーム仕様、新しい社会的事象など）を**最低1つ**導入する
- その新規設定は「背景の飾り」ではなく、**物語の核心的展開に不可欠な要素**であること

出力する前に以下の配下の世界観とゲーム設定をくまなく探索し、`sao-novel/CLAUDE.md`の条件に満たす内容にすること

　/Users/ishikawaryuusuke/Desktop/aidev/sao-novel/world

### 出力形式

```markdown
## 核（コア・コンセプト）
【一文】この小説は○○の話である。

### リアル感の関与
- 三層構造のうち関与する層とその関わり方

### ゲーム性の関与
- 物語を駆動するゲームシステム要素とその役割

### 舞台
- 何層のどこが舞台か(設定に存在しないものを各層の特徴とかに基づいて作成しても良い)

### 新規設定
- 導入する新規設定の概要
- それがシナリオの核にどう関わるか
```

## Step 2：プロット骨格の設計

### やること

核をもとに、物語の**始点→転換点→終点**の骨格を設計する。

### 設計要件

#### 面白さの担保

| 要素 | 具体的に何を設計するか |
|------|----------------------|
| **意外性** | 読者の予測を裏切る展開を**最低2箇所**設計する。「そう来るか」と思わせるもの。ただし後から振り返れば伏線があったとわかる構造にする |
| **伏線と回収** | 序盤〜中盤に仕込む伏線と、その回収ポイントを**明示的に対応表で設計**する。回収は自然であること（こじつけ感がない） |
| **テンポ** | 緊張と緩和のリズムを意識する。ずっと緊迫でもずっと日常でもない。メリハリの設計 |

#### リアル感とゲーム性のプロットへの組み込み

- プロットの各転換点で、「リアル感の三層構造のどの層が機能しているか」を明示する
- ゲームシステムがプロットに関与する箇所を明示する
- 新規設定が初出する箇所・機能する箇所・回収される箇所を明示する

出力する前に以下の配下の世界観とゲーム設定を探索し、`sao-novel/CLAUDE.md`の条件に満たす内容にすること
  /Users/ishikawaryuusuke/Desktop/aidev/sao-novel/settings
　/Users/ishikawaryuusuke/Desktop/aidev/sao-novel/world

### 出力形式

```markdown
## プロット骨格

### 始点（状況設定）
- 誰が / どこで(どの層の話か＝Map情報をもとに事実に則した上で場所自体は創作しても良い、層を渡ってもOK) / 何をしている状態から始まるか
- 読者に最初に提示する情報は何か

### 展開（中盤の主要イベント）
- 転換点1：[内容] ← リアル感の層: / ゲーム性:
- 転換点2：[内容] ← リアル感の層: / ゲーム性:
- ...

### 終点（結末）
- 物語がどこに着地するか
- 核（コア・コンセプト）に対する回答になっているか

### 伏線対応表
| # | 伏線（設置箇所） | 回収（回収箇所） | 効果 |
|---|-----------------|-----------------|------|
| 1 | ... | ... | ... |

### 意外性ポイント
| # | 箇所 | 読者の予測 | 実際の展開 | なぜ自然か |
|---|------|----------|-----------|----------|
| 1 | ... | ... | ... | ... |
```

## Step 3：キャラクター設計

### やること

プロットを成立させるために必要なキャラクターを設計する。

### 設計の原則

- キャラクターは**リアル感とゲーム性を物語上で成立させるための要素**である
- しかし道具ではない。以下の最低限の人物造形を行う

### 各キャラクターに定義する項目

| 項目 | 内容 |
|------|------|
| **名前** | ゲーム内アバター名（必要に応じて現実名も） |
| **立場** | 攻略組/中層/定住者/レッドなど、社会的位置づけ |
| **目的** | この時点で何を目指しているか（攻略？生存？復讐？日常の維持？） |
| **性格の核** | 一言で表せる性格特性（慎重、楽天的、合理主義、等） |
| **認知の偏り** | 「ゲームだ」寄りか「現実だ」寄りか、その揺れ方の特徴 |
| **新規設定との関わり** | 新規設定にどう関与するか |

### 注意

- 主人公が無双する構造は禁止（CLAUDE.md禁則）
- 全キャラクターがプロットに機能的に関与すること。飾りキャラは作らない

## Step 4：章構成の設計

### やること

プロット骨格をもとに、実際の章分割を行う。

### 章設計の目的

**読者がSAOの世界観・設定を自然に理解しながら、面白い流れで読み進められる構成を作る。**

章は「形骸的な区切り」で構わない。「ここからここまでで、何を読者に伝え、何を体験させるか」の指針。

### 各章に記載する項目

```markdown
### 第N章：[仮タイトル]

- **この章の目的**：読者に何を伝える/体験させるか
- **主要イベント**：何が起きるか（箇条書き）
- **伏線の設置/回収**：この章で仕込む伏線 / 回収する伏線（伏線対応表の#を参照）
- **ゲーム性の要素**：この章で登場するゲームシステム要素
- **テンポ**：緊張度（高/中/低）とその理由
- **章末の引き**：次章への推進力になる要素
```

### 章構成の全体設計指針

- **序盤（全体の約20%）**：世界観と日常を提示しつつ、核心への伏線を静かに仕込む
- **中盤（全体の約50%）**：問題の顕在化→エスカレート→意外な転換。テンポの緩急が最も重要な区間
- **終盤（全体の約30%）**：伏線回収→クライマックス→着地。核に対する回答を提示する

## Step 5：壁打ち評価と改善

### やること

Step 1〜4の成果物を**Agentに壁打ち評価させ**、必要に応じて修正する。

### 評価の実行方法

Agentツール（subagent_type: general-purpose）を起動し、成果物の全文をプロンプトに含めること。

レビューの6観点・配点・出力形式・判定基準はすべて以下の独立ファイルに定義：

**`agents/concept-reviewer.md`**

このファイルの内容をサブエージェントのプロンプトに含めること。

### 呼び出し方

```
Agent(
  subagent_type: "general-purpose",
  prompt: agents/concept-reviewer.md の内容 + レビュー対象の成果物全文
)
```

### 評価結果の扱い

| 合計点 | 対応 |
|--------|------|
| 80以上 | ユーザーに提示し、フィードバックを求める |
| 60〜79 | 問題点を修正し、再度評価にかける（最大2回） |
| 59以下 | Step 1（核）から再設計する |

### 改善のループ

1. Agent評価で指摘された問題点を確認する
2. 該当するStep（核/プロット/キャラ/章構成）に戻って修正する
3. 修正箇所から影響を受ける後続Stepも更新する
4. 再度Agent評価にかける
5. 基準を満たしたらユーザーに提示する
