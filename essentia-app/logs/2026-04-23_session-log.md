## 1. Essentia ライセンス問題

### ⭐ 出典明記では商用利用できない
- **Essentiaライブラリ（AGPL v3）**：ネット越しにサービス提供する時点でソース全開示義務。出典表記では不十分。
- **モデルファイル（CC BY-NC-ND 4.0）**：NC＝非商用のみ。出典を書いても商用禁止は変わらない。
- MTGへの商用ライセンス申請は可能だが、条件・費用は非公開。ライブラリ・モデルの**両方**の許諾が必要。

### ⭐ 現状のパフォーマンス限界

| 条件 | 時間 |
|------|------|
| CPU 2vCPU / 30秒音声 / 1リクエスト | 3〜8秒 |
| GPU T4 / 30秒音声 / 1リクエスト | 0.5〜1秒 |
| Cloud Run コールドスタート | 追加30〜60秒 |

- **現実装は同時リクエスト非対応**（キューなし）→ 同時5人で最大40秒待ち
- CPU環境で商用スケールは非現実的

### モデル構成
- Stage1：`discogs-effnet-bs64-1.pb`（EfficientNet B4ベース、約100MB）→ 1280次元embedding
- Stage2：`genre_discogs400-discogs-effnet-1.pb` → 400クラス分類
- 入力：16kHz mono

## 2. ACRCloud 調査結果

### 基本性能
- 最短 **3〜5秒**の音声から認識可能
- フィンガープリント精度：98%以上、MIREX 2015・2016で1位

### ⭐ ジャンルデータの構造

| 層 | 内容 | 備考 |
|----|------|------|
| デフォルト（自社DB） | `genres: [{ name: "Pop" }]` | マイナー曲は空になりがち |
| Spotify/Deezer連携 | IDのみ返却、詳細は各API別途コール | デフォルト有効 |
| Music Story（有料アドオン） | 800種類以上、一次・二次ジャンル | 精度・網羅性が最高 |

### ⭐ ジャンル精度の実態
- 独立したジャンル精度ベンチマークは存在しない
- 開発者の不満は「間違っている」ではなく「**空で返ってくる**」
- メジャー曲：問題なし。マイナー・地域特有の楽曲：カバレッジ弱い
