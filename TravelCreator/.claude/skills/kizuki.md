# 気付きスキル — 処理フロー

## Step 0: 国と場所を確認する（必須）

入力に「国」と「場所（都市・街・遺跡など）」の**両方**が含まれているか確認する。

- **どちらか片方でも欠けていたら、処理を止めて次の形式で聞き返す：**

```
国と場所を教えてください。
例）エジプト・カイロ、ヨルダン・ペトラ
```

- 両方が明示されたら Step 1 に進む。
- 国は Egypt / Jordan のどちらかに対応付ける。
- 場所は都市・街・遺跡・地区など具体的な地名であれば何でも可。

## Step 1: 入力を解析する
- Step 0 で確認した国（Egypt / Jordan）とカテゴリを特定する
- どのカテゴリに関係するか（history / religion / culture / food / practical / geography / economy / overview）を判断する
- 複数カテゴリにまたがる場合はすべて対象にする

## Step 2: CountryKnowledgeから関連ファイルを読み込む
- 特定した国・カテゴリのファイルのみを読む（不要なファイルは読まない）
- ファイルが存在しない場合はスキップ（エラーにしない）
- 読み込んだ内容をもとに関連情報を抽出する

## Step 3: Thoughtファイルを作成・出力する

ファイル名: `Thought/YYYY-MM-DD_場所_topic.md`
- 日付は今日の日付を使う
- 場所は Step 0 で確認した地名を使う（例: Cairo, Petra, WadiRum）
- topicはユーザー入力から3〜5語で命名する（日本語OK）
- 同日・同場所で複数ある場合は `_2`, `_3` を付ける

ファイルの構成:
```
# [日付] [場所] トピックタイトル

## 観察・疑問（要約）
ユーザーの入力を1〜3文で整理・要約する。
元の言葉のニュアンスを保ちつつ、疑問の核心を明確にする。

## Claudeの考察
この疑問・観察に対するClaudeの分析・解釈を書く。
- 背景にある文化・歴史・宗教的文脈
- なぜそう感じたのかの説明
- 深掘りすると面白い視点
CountryKnowledgeの情報を根拠として活用する。
CountryKnowledgeにない知識を使う場合は「※」を付けて明記する。

## 参照した知識（CountryKnowledge）
- `CountryKnowledge/Egypt/history.md` — 参照した具体的な項目
- （読んだファイルを列挙。読んでいない場合は「なし」）

## 関連知識メモ
CountryKnowledgeから抽出した関連情報の要点を箇条書きで。
ユーザーが後で読み返したときに役立つ補足情報を添える。

## この気付きから得た学び
この気付きを通じて何を理解したか、一目でわかるように箇条書きで簡潔にまとめる。
- 〇〇という事実・視点を新たに知った
- 〇〇という誤解が解けた / 見方が変わった
- 〇〇と〇〇の関係・つながりが見えた
（該当するものを2〜4点に絞って記載。冗長にしない）
```

## Step 4: CountryKnowledgeに知識を追記する（必要な場合）
- Step 3でCountryKnowledgeにない知識を使った場合、該当する国・カテゴリのファイルに追記する
- ファイルが存在しない場合は新規作成する
- 追記形式: 箇条書き、情報源や文脈を簡潔に添える

## Step 5: 完了を伝える
Thoughtファイルのパスをユーザーに伝える。一言で。
```
→ Thought/2026-04-22_Petra_ナバテア人とローマ.md に出力しました。
```

## 禁止事項
- 国・場所の確認（Step 0）以外の確認を挟まない
- 長い前置きや説明を返さない
- Thoughtへの出力を省略しない
- CountryKnowledgeを丸ごと読もうとしない（必要なファイルのみ読む）
