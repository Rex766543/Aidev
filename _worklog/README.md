# _worklog

重要タスク実行時の「何を・なぜ・どう決めたか・ハマり所・残タスク」を、後から見返せる形で残す作業ログ。

## 性質
- **起動時に自動ロードしない。** コンテキストを普段は消費しない。
- 必要なとき（過去の判断を思い出したいとき）に Claude が該当ファイルだけ読む。
- 完全な生トランスクリプトは `~/.claude/projects/<project>/*.jsonl` に自動で残っているので、ここには**要約だけ**置く。

## 書き方
- ユーザーが「logに入れて」等と指示したとき、または重要タスク完了時に追記する（`/log` コマンド）。
- 1タスク＝1ファイル: `YYYY-MM-DD_<タスク名>.md`。
- 中身は人が読める要約（決定理由・つまずき・残件）。逐語コピーはしない。
